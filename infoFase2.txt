A ideia: Asymmetric Growth
A hipótese é esta:

A fronteira de decisão entre classes é muito mais complexa nas regiões onde a minoria existe. A árvore standard usa os mesmos critérios de paragem em todo o lado — mas devia ser mais "paciente" e continuar a crescer nos nós que ainda contêm minoria.

Em termos práticos: quando o train() está a decidir se para ou continua, verifica o rácio local de minoria e compara com o rácio global. Se o nó tem proporcionalmente mais minoria do que o dataset completo, os critérios de paragem ficam mais relaxados.

Os 3 critérios de paragem que vamos tornar assimétricos
No train() atual existem estas condições de paragem:
python# Condição 1 — profundidade máxima atingida
current_depth >= self.max_depth

# Condição 2 — amostras a menos para fazer split
n_samples < self.min_samples_split

# Condição 3 — gain demasiado baixo
gain < self.minimum_gain
Cada um destes vai ter um valor diferente consoante a concentração local de minoria:

Condição 1 — max_depth assimétrico
python# Nó normal:   para em max_depth = 5
# Nó com muita minoria: pode ir até max_depth + minority_extra_depth

minority_ratio_local  = np.sum(y == 1) / len(y)
minority_ratio_global = self.priors.get(1, 0.5)

# Fator: o quanto este nó está "enriquecido" de minoria
enrichment = minority_ratio_local / (minority_ratio_global + 1e-9)
# ex: local=20%, global=10% → enrichment = 2.0
# ex: local=5%,  global=10% → enrichment = 0.5

# Profundidade efectiva: nós enriquecidos de minoria recebem "crédito" extra
effective_max_depth = self.max_depth + int(enrichment > 1.5) * self.minority_extra_depth

Condição 2 — min_samples assimétrico
python# Nó normal:         para se n < 10
# Nó rico em minoria: para se n < 4  (aceita nós menores)

if enrichment > 1.5:
    effective_min_samples = max(2, self.min_samples_split // self.asymmetry_factor)
else:
    effective_min_samples = self.min_samples_split

Condição 3 — minimum_gain assimétrico
python# Nó normal:         precisa de gain >= 0.01
# Nó rico em minoria: aceita gain >= 0.001  (10x mais sensível)

if enrichment > 1.5:
    effective_min_gain = self.minimum_gain / self.asymmetry_factor
else:
    effective_min_gain = self.minimum_gain

O parâmetro enrichment — porquê este cálculo
O enrichment mede: "este nó tem mais minoria do que seria esperado pelo acaso?"
enrichment = local_minority_ratio / global_minority_ratio

enrichment = 1.0  →  nó representa fielmente o dataset   → comportamento normal
enrichment = 2.0  →  minoria 2x concentrada aqui         → crescer mais
enrichment = 5.0  →  minoria muito concentrada           → crescer bastante mais
enrichment = 0.5  →  minoria sub-representada            → comportamento normal ou mais restrito
É uma medida relativa — não depende do valor absoluto do imbalance. Funciona da mesma forma se a minoria for 5% ou 30%.
